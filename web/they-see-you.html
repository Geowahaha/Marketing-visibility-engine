§ the unsigned problem

User-Agent คือการอ้าง ไม่ใช่หลักฐาน

A User-Agent is a claim, not proof

ทุกแถวในตารางด้านบน ยกเว้นแถว AIBotAuth คือ request ที่ส่ง User-Agent string ที่ใครก็ตั้งค่าได้ เซิร์ฟเวอร์ของคุณไม่มีทางรู้ว่าเป็น GPTBot ตัวจริงหรือแค่ใครบางคนที่พิมพ์สตริงนั้นลง

Every row in the table above — except AIBotAuth — is a request carrying a User-Agent string any script can set. Your server has no way to know if that's the real GPTBot or someone who just typed that string in.

  1. AIBotAuth เซ็นทุก request ด้วย Ed25519 ตาม RFC 9421 (Web Bot Auth) — ลายเซ็นผูกกับโดเมน ช่วงเวลา และ key identity ของเรา AIBotAuth signs every request with Ed25519 under RFC 9421 (Web Bot Auth) — the signature is bound to the target domain, a time window, and our key identity
  2. เซิร์ฟเวอร์ของคุณดึงกุญแจสาธารณะของเราจาก /.well-known/http-message-signatures-directory และตรวจสอบลายเซ็นได้ทันที — ไม่ต้องมีผู้กลาง Your server fetches our public key from /.well-known/http-message-signatures-directory and verifies the signature inline — no intermediary needed
  3. หากลายเซ็นผ่าน แสดงว่าเป็น AIBotAuth ตัวจริง ไม่ใช่ใครก็ตามที่คัดลอก User-Agent ของเรา — ปลอมแปลงทางคณิตศาสตร์ไม่ได้ If the signature verifies, it's the real AIBotAuth — not anyone who copied our User-Agent string. It's mathematically unforgeable
